Overview

CurlPlan is a local first personal curling app that puts your calendar, game prep, practice logs, and rink memory in one place. There is no account to create, no server to connect to, and no subscription. Open the page, and you are ready to go.

Your data lives in your browser's local storage. That means it stays on your device, loads instantly, and works offline once the page is cached. You can export everything as a JSON file at any time for backup or transfer.

What CurlPlan Does

  • Calendar Queue: Track upcoming league nights, bonspiels, practices, and games. Filter by type and search by keyword.
  • Game Planner: Build a per date prep sheet with pre game goals, post game reflection, scores, and a customizable checklist that carries forward.
  • Game Log: Record scores, shot percentages, opponents, key shots, and position. Results auto compute from scores.
  • Practice Log: Track drill mix, session length, focus, and notes. A running summary shows which shots you practice most.
  • Ice Notes: Log speed, curl, and rink specific observations. Build a rink memory over time so you know what to expect at each club.
  • Season Stats Dashboard: Six auto computed insight cards (record, best position, score differential, streak, rink split, shot trend) with sparkline charts and configurable time ranges.
  • Print Reports: Generate a printable game report that pulls in planner goals, reflection, checklist state, and matching ice notes for any game date.
  • UI Issue Tracker: Log and manage internal bugs and feature requests using P1/P2/P3 severity with status tracking.

Tools Needed

CurlPlan runs entirely in your browser. Here is what you need:

  • A modern browser. Chrome, Firefox, Safari, or Edge. Anything released in the last few years will work.
  • The CurlPlan page. Open index.html directly from disk, or serve it through GitHub Pages or any static host.
  • That's it. No install, no dependencies, no internet connection after the first load.
Good to Know

Your data stays in the browser that created it. If you switch browsers or clear site data, you will lose your entries unless you export first. The Export button is your backup.

Step by Step Instructions

Getting Started

1
Open CurlPlan. Load index.html in your browser. The app starts on the Dashboard with demo data already loaded so you can explore every feature immediately.
2
Explore the demo data. Click through the tabs: Dashboard, Calendar, Game Planner, Game Log, Practice, Ice Notes, UI Issues. The demo set includes sample events, games, practice sessions, ice readings, and planner entries so every view has something to show.
3
Clear the demo when ready. When you want to start fresh, click Reset Demo Data in the top bar (this restores the demo), or export your real data first, then import it into a clean session.

Adding an Event

1
Click "+ Quick Add Event" or navigate to Calendar and click "+ Add Event." The event modal opens.
2
Fill in the name and date. These two fields are required. Everything else (time, type, position, location, sheet, notes) is optional but helpful.
3
Pick a type. Choose League, Game, Practice, Bonspiel, or Other. This controls how the event appears in the Calendar filter.
4
Click "Save Event." The event appears in your Calendar and shows up on the Dashboard if it is upcoming. A status message and toast confirm the save.

Logging a Game

1
Open the Game Log tab and click "+ Log Game."
2
Enter date and opponent. These are required. Then fill in scores (Our Score and Their Score). CurlPlan auto computes the result: higher score wins, equal scores draw.
3
Add shot percentage if you tracked it. This is optional, but logging it for at least two games unlocks the shot trend sparkline on the Dashboard and Game Log.
4
Record position, rink, key shot, and notes. Position and rink feed into the Season Stats Dashboard (best position, top rink). Key shot and notes appear in the game row and in the printable report.
5
Click "Save Game." The game appears in the results table and the Dashboard updates immediately.

Using the Game Planner

1
Open the Game Planner tab. The planner shows today's date by default. Use the Previous/Next buttons to navigate to any date.
2
Fill in the Pre Game section. Start time, rink, opponent, position, goals, and ice notes. The checklist at the bottom is your operational prep list (equipment, warmup, communication).
3
Customize your checklist. Add items with the text input and "Add Item" button. Reorder with arrow buttons. Remove items you do not need. Click "Reset Defaults" to go back to the original three items. Your custom list carries forward to new dates.
4
After the game, fill in the Post Game section. Scores, key shot, and reflection. This is where you capture what worked, what missed, and what to keep next time.
5
Click "Save Notes." The planner entry is stored for that date. Jump to other dates using the Recent Planner Entries cards at the bottom without losing your saved work.

Logging Practice and Ice Notes

1
Practice: Open the Practice tab, click "+ Log Practice." Pick a date, duration, check the drills you worked (Draw, Guard, Takeout, Freeze, Hit & Roll, Broom Work), and write your focus and notes. Save.
2
Ice Notes: Open the Ice Notes tab, click "+ Add Ice Notes." Pick a date, name the rink, tap the speed dots (1 to 5), choose a curl type (Very Straight through Big Finish), and describe the ice. Save. The Rink Memory panel on the right builds over time.

Printing a Game Report

1
Go to Game Log. Find the game you want and click "Print" in the Actions column.
2
CurlPlan generates a full report. It pulls in the game score, key shot, and notes. It also includes the planner entry for that date (goals, reflection, checklist state) and any matching ice notes. Your browser's print dialog opens automatically.
3
Print or save as PDF. The report is formatted for letter size. After printing, the report overlay clears automatically.

Exporting and Importing Data

1
Export: Click the "Export" button in the top bar. A file named curlplan-data.json downloads to your device. This contains all your events, games, practice, ice notes, issues, and planner entries.
2
Import: Click "Import," select a CurlPlan JSON file, and the app loads it. This replaces all current data, so export first if you want to keep what you have.

Using Season Stats

1
The Season Stats Dashboard is on the main Dashboard view. It computes six insight cards from your game log: Record, Best Position, Average Score Differential, Longest Win Streak, Top Rink Split, and Shot % Trend.
2
Change the time range. Click "This Season" (current calendar year), "All Time" (every game), or "Last 10" (most recent 10 games). The cards and sparkline charts update instantly.
3
The more games you log, the richer the stats. Position data requires logging your position on games. Shot trend requires at least two games with shot percentage recorded.

Tips

Export Often

Your data lives in browser storage. If you clear your browser data, switch devices, or use a different browser, your entries will not be there. Click Export after every session to keep a backup file. It takes one second.

Use the Planner Before and After

Fill in your pre game goals and checklist before you leave for the rink. After the game, come back and fill in scores, key shot, and reflection while the details are still fresh. The printable report ties both halves together.

Log Position and Rink on Games

The Season Stats Dashboard computes win percentage by position and by rink. These insights only work if you include position and rink when you log games. The extra five seconds per entry pays off across a full season.

Track Shot Percentage

Shot percentage is optional, but logging it for at least two games unlocks the shot trend sparkline on both the Dashboard and the Game Log performance panel. If your club tracks individual stats, enter yours here.

Use Hash URLs to Jump Straight to a View

Bookmark index.html#planner or index.html#games to open CurlPlan directly on the view you use most. Valid hashes: #dashboard, #calendar, #planner, #games, #practice, #ice, #issues.

Customize the Planner Checklist Once

The default checklist has three items: Equipment check, Warm up routine, Team communication reminders. Customize it once with your actual prep steps (brooms, shoes, stopwatch, water, team text). Your custom list carries forward to all future dates automatically.

Use Keyboard Shortcuts

Press Esc to close any open modal without saving. Press Enter in the checklist input to add an item without clicking the button. Click outside a modal to dismiss it.

Common Mistakes

Not Exporting Before Clearing Browser Data

CurlPlan stores everything in browser local storage. Clearing site data, resetting the browser, or switching to a new device means your entries are gone unless you exported them. Make export part of your routine.

Forgetting to Click "Save Notes" in the Planner

The planner does not auto save. If you fill in fields and then navigate to another date or tab without clicking "Save Notes," your entries for that date are lost. Always save before navigating away.

Importing Without Exporting First

Import replaces all current data with the contents of the imported file. If you have unsaved work or entries you want to keep, export your current data before importing anything.

Leaving Scores Blank and Expecting a Result

The W/L/D result badge auto computes from the Our Score and Their Score fields. If you leave both blank, no result appears. Enter both scores and the result populates automatically.

Skipping Position and Rink on Game Entries

The Season Stats Dashboard breaks down win rate by position and by rink. If you skip these fields, those insight cards show "Unknown" or have no data to compute. Fill them in consistently for the best analytics.

Expecting Data to Sync Across Devices

CurlPlan is local first. There is no cloud sync. To move data between devices, export on one device and import on the other. The JSON file is the bridge.

Clicking "Reset Demo Data" By Accident

This replaces your current data with the demo set. A confirmation dialog appears first, but if you click through it too fast, your entries are overwritten. If this happens and you have an export file, import it to restore.
